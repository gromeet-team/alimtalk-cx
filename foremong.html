적용 개선 원칙 (7가지)
원칙 1

메시지 간소화

결제완료 시점의 인증 정보 5종 → 배송시작/배송완료로 분산. 각 시점에 맞는 정보만 전달.

원칙 2

버튼 CTA 우선순위 통일

1번: 핵심 행동(정품등록/리뷰/조회) → 2번: 보조 행동 → 3번: 채널추가. 전 캠페인 동일 적용.

원칙 3

구매확정 ↔ 리뷰 분리

구매확정 유도 시점에서 리뷰 요청 제거. 구매확정만 집중 → +7일 캠페인에서 리뷰 전담.

원칙 4

설치확인 터치포인트 추가

배송완료 +1~2일 "설치 잘 되셨나요?" 메시지 신설. 초기 불량/설치 실수 CS 선제 대응.

원칙 5

배송완료 정품등록 CTA 복원

변경요청에 있던 정품등록 버튼이 작업본에서 누락 → 1번 버튼으로 복원.

원칙 6

시즌 대응 (한달 리뷰)

"겨울이 끝나갈 무렵" 등 시즌 고정 표현 제거 → 시즌 중립 or 분기 처리 가능한 구조로.

원칙 7

성과 추적 체계

캠페인별 오픈율/클릭율/전환율 추적 구조 설계. 월 1회 리뷰로 데이터 기반 개선.

Phase 1 (즉시): 통합 메시지 유지하되, 톤 개선
→ "혹시 저희가 도울 수 있는 부분이 있을까요?"

Phase 2 (시스템 확인 후): 사유별 분기 적용
→ 단순변심: 다른 사이즈/모델 제안
→ 배송지연: 예상 도착일 + 사과 쿠폰
→ 서비스불만: CS 직통 연결

취소완료 — 윈백 장치 추가

기존 의견 수집 구글폼 유지 (Good)
+ "다음 시즌에 다시 찾아주시면
특별 할인을 드리겠습니다" 윈백 메시지

교환 완료 — 재만족 확인

교환 완료 후 +3일에
"교환 제품은 만족스러우신가요?" 추가 터치
→ 리뷰 유도 연결
